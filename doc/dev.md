# ⏰ time capsule

# 🧊前言

`time capsule`程序的功能是用来管理用户的文件数据。

`time capsule`是一个时间胶囊的比喻，表示这个工具可以保存和管理文件，并且记录这些文件的操作日志。🎈

在数字化的时代，我们有很多数字资产，如何管理维护这些数字化资产，这是一个问题，而 `time capsule`就是为了解决这个问题而诞生的。💪

# 🧊功能特性

`time capsule`的功能特性如下：

- 轻量级，无需安装，无需配置，直接运行即可， 删除即卸载。
- 支持web端 和桌面端。
- 用户自定义存储路径
- 不入侵用户原始文件数据（即不会对用户文件数据做任何修改），除非用户主动删除，复制，移动。
- 工程文件随时删除，随时备份。
- 支持文件的标签管理。
- 支持查看文件的详细信息。
- 支持文件多种条件搜索。
- 支持文件/文件夹的复制，移动，删除操作。
- 文件变更的日志操作记录跟踪。

timeCapsule主要是扫描用户添加的文件夹，把这些文件和文件夹数据的信息提取出来，例如文件名、大小、修改时间等信息，并把这些信息保存到数据库中。同时，timeCapsule还会为这些文件生成缩略图，以便用户可以快速查看文件内容。📂

程序运行的配置文件和中间生成文件全部在 `appData`目录下，可以理解这个目录就是工程文件目录。这个目录不会影响到用户的文件数据文件夹，可以随时删除卸载，且也不会再用户的数据文件夹下生成其他文件。即 `非侵入式`。即使用户把appData删除了，也不会影响到用户的文件数据文件夹。👍

默认配置中，`timeCapsule`运行所需的数据，在程序同级目录的 `assets`目录下。目前不支持修改。不可以删除。
`assets`目录:

```
www --------------- web资源，直接拷贝过来即可。🌐
ffmpeg.exe -------- 视频信息处理🎥
ffplay.exe -------- 视频播放📺
```

`timeCapsule`运行产生的数据，在程序同级目录的 `appData`目录下。目前不支持修改。可以直接删除。

`appData`的文件结构如下：

```

log ---------------- 存放日志文件📝
thumbnail ---------- 存放缩略图🖼️

trash -------------- 文件回收站🗑️
```

### ⚠️存储备份功能

文件备份是指把一个存储中的文件备份到另一个存储中。
备份的操作就是把一个存储中的文件复制到另一个存储中。
同时也把 `appData`目录拷贝过去。
但这时，去打开备份目录是取法工作的，因为 `appData`目录下的一些配置文件，数据库文件等都是针对当前存储设计的，程序是会到当前存储中寻找文件，而不是从备份存储中寻找文件。所以对于备份存储是无效的。💾

### ⚠️存储恢复功能

1，先停止程序。⏹️
2，直接手动把备份存储拷贝到当前存储中。例如可以把 `dirA_bck1`拷贝到 `dirA`中；
3，同时也把 `appData`目录拷贝到与后台app.exe同级的目录下。📂
4，启动程序。▶️
这时即可恢复到备份的状态。🔙

注意：
由于appData目录下记录的是多个存储的数据，如果在存储恢复的过程中。虽然把 `dirA_bck1`拷贝到 `dirA`中，也把 `appData`目录拷贝到与后台app.exe同级的目录下。但是 `dirB`中的文件可能会与 `appData`中的记录不一致。

如下操作情景就会出现数据紊乱的情况：

1. 备份 `DirA`到 `dirA_bck1`。
   这时，`appData`也拷贝到了备份目录。`appData`中记录的还有 `dirB`的信息。📝
2. 修改 `dirB`中的一些内容。
   这时 `appData`中关于 `dirB`的信息与 `dirA_bck1`中记录的信息已经不一致。❌
3. 恢复 `dirA_bck1`到 `dirA`。
   这时，有两种情况：
   - <br />
     1. 把 `appData`目录也拷贝到与后台app.exe同级的目录下。
        这时，拷贝过来的 `appData`中记录的 `dirB`的信息。就与当前的 `dirB`的信息不一致。❌
   - <br />
     1. 不拷贝 `appData`目录。
        这时，`appData`中记录的 `dirB`的信息。就与当前的 `dirB`的信息一致。
        但是，`appData`中记录的 `dirA`的信息，可能与备份过来的 `dirA`的信息不一致。❌
   所以我们建议还是只添加一个存储。以便使用备份功能时数据紊乱。💡
